# 快速开始指南

## Windows 用户（3 步启动）

### 第 1 步：配置 API Key

编辑 `set-env.bat` 文件：
```cmd
notepad set-env.bat
```

找到这一行，填入你的 API Key：
```cmd
set OPENAI_API_KEY=sk-your-api-key-here
```

如果使用代理，还需要修改：
```cmd
set OPENAI_BASE_URL=https://your-proxy.com/v1
```

保存并关闭文件。

### 第 2 步：运行配置脚本

双击运行 `set-env.bat`，或在命令行中执行：
```cmd
set-env.bat
```

你会看到：
```
✓ 环境变量配置完成！

已配置的环境变量：
  OPENAI_API_KEY = sk-your-api-key-here
  OPENAI_BASE_URL = https://api.openai.com
  ...
```

### 第 3 步：启动项目

**在同一个命令行窗口中**运行：
```cmd
start-project.bat
```

选择要启动的项目（1-7），然后等待启动完成！

## Linux/Mac 用户（3 步启动）

### 第 1 步：配置 API Key

```bash
# 复制配置模板
cp .env.example .env

# 编辑配置文件
nano .env  # 或使用 vim、code 等编辑器
```

填入你的配置：
```bash
OPENAI_API_KEY=sk-your-api-key-here
OPENAI_BASE_URL=https://api.openai.com  # 可选
```

### 第 2 步：加载环境变量

```bash
export $(cat .env | xargs)
```

### 第 3 步：启动项目

```bash
chmod +x start-project.sh
./start-project.sh
```

## 常见问题

### Q1: 环境变量配置后不生效？

**Windows**: 确保在运行 `set-env.bat` 的**同一个命令行窗口**中启动项目。环境变量只在当前窗口有效。

**Linux/Mac**: 确保使用 `export` 命令加载环境变量。

### Q2: 如何验证配置是否正确？

运行 `set-env.bat` 后，会显示当前配置的环境变量。检查 `OPENAI_API_KEY` 是否正确。

### Q3: 如何使用代理？

在 `set-env.bat` 中设置：
```cmd
set OPENAI_BASE_URL=https://your-proxy.com/v1
```

### Q4: 如何使用 GPT-4？

在 `set-env.bat` 中设置：
```cmd
set OPENAI_MODEL=gpt-4
```

### Q5: 端口 8080 被占用怎么办？

在 `set-env.bat` 中修改端口：
```cmd
set SERVER_PORT=8081
```
